# Triage Hints: LoRa (Chirp Spread Spectrum)

These hints help the LLM identify and verify Semtech LoRa CSS signals from statistical reports generated by `triage_iq.py`.

---

## 🔍 Key Spectral Indicators

### 1. Occupied Bandwidth (OBW)
LoRa signals are strictly limited to standard channel bandwidths. Look for these 99% OBW values:
* **Sub-GHz LoRa**: Strictly **125 kHz, 250 kHz, or 500 kHz**.
* **2.4 GHz LoRa**: **200 kHz, 400 kHz, 800 kHz, or 1.6 MHz**.
* If the bandwidth is wide (e.g. 5 MHz, 10 MHz), it is not a standard LoRa signal.

### 2. Low PAPR (Peak-to-Average Power Ratio)
* Like FM, CSS is a constant-envelope phase-modulation technique.
* The PAPR of a LoRa transmission is **low (~1.5 dB to 3.0 dB)**. This is a crucial differentiator: it rules out high-PAPR multi-carrier systems like OFDM (8-12 dB) and narrow multi-tone signals.

### 3. OFDM Flatness Score
* LoRa does not use orthogonal subcarriers; it sweeps a single carrier. The flatness score will be **low to medium (< 0.5)**.

---

## ⏱ Key Temporal Indicators

### 1. Packet Duration & Duty Cycle
* LoRa is used for low-power sensor reporting, so the duty cycle is **very low (< 1%)**.
* Packets are relatively slow due to the spreading factor. Active burst durations range from **10 ms to over 1.5 seconds**. 
* If you see active bursts shorter than 5 ms, they are likely not LoRa.

---

## 🧬 Autocorrelation Periodicities (Symbol Duration)

During the preamble of a LoRa transmission, the signal repeats identical unmodulated up-chirps. This creates strong correlation peaks at lags matching the symbol duration:
$$T_{sym} = \frac{2^{SF}}{BW}$$

### Symbol Duration Reference (ms):

| Spreading Factor (SF) | 125 kHz BW | 250 kHz BW | 500 kHz BW |
|---|---|---|---|
| **SF7** | 1.024 ms | 0.512 ms | 0.256 ms |
| **SF8** | 2.048 ms | 1.024 ms | 0.512 ms |
| **SF9** | 4.096 ms | 2.048 ms | 1.024 ms |
| **SF10** | 8.192 ms | 4.096 ms | 2.048 ms |
| **SF11** | 16.384 ms | 8.192 ms | 4.096 ms |
| **SF12** | 32.768 ms | 16.384 ms | 8.192 ms |

### Example Lag Verification:
If the triage report has a sample rate of **2.0 MSPS** and detects a LoRa SF7 125 kHz signal:
* Symbol duration $= 1.024\ \text{ms}$.
* Expected sample lag $= 1.024\ \text{ms} \times 2000\ \text{samples/ms} = \mathbf{2048\ \text{samples}}$.
* If the report shows a strong autocorrelation peak at exactly **2048 samples**, it strongly confirms **LoRa SF7**.
